# 火箭回收算法设计总结

本文档总结了火箭垂直着陆回收算法的设计和实现。算法基于最优控制理论和PID控制，模拟SpaceX Falcon 9火箭的垂直着陆过程。

## 算法设计

### 1. 数学模型

#### 状态变量
```
状态向量: X = [x, y, vx, vy, θ, ω, m, fuel]
```
- `x, y`: 位置坐标 (m)
- `vx, vy`: 速度分量 (m/s)
- `θ`: 姿态角 (弧度)
- `ω`: 角速度 (弧度/s)
- `m`: 总质量 (kg)
- `fuel`: 燃料质量 (kg)

#### 控制变量
```
控制向量: U = [throttle, gimbal]
```
- `throttle`: 节流阀 [0, 1]
- `gimbal`: 摆角 [-10°, 10°]

#### 动力学方程

**平动方程:**
```
ax = (F * sin(θ + g)) / m
ay = (F * cos(θ + g)) / m - g
```

**转动方程:**
```
α = (F * g * L/2) / I
```

**质量变化:**
```
dm/dt = -F / (Isp * g0)
```

### 2. 控制策略

#### 2.1 PID控制器

**水平控制:**
```python
desired_theta = -Kp_x * dx - Kd_x * dvx
```
- 通过姿态角调整水平推力
- 水平位置误差驱动姿态角变化

**垂直控制:**
```python
base_thrust = mass * g + Kp_y * dy + Kd_y * dvy
throttle = base_thrust / thrust_max
```
- 通过节流阀调整推力大小
- 垂直位置和速度误差驱动推力变化

**姿态控制:**
```python
gimbal = (desired_theta - actual_theta) * Kp_theta
```
- 通过摆角调整推力方向
- 姿态角误差驱动摆角变化

#### 2.2 分段控制策略

**远程阶段 (高度 > 1000m):**
- 目标: 快速接近目标位置
- 策略: 大角度调整，快速减速
- 控制: 高增益PID，大推力

**中程阶段 (100m < 高度 ≤ 1000m):**
- 目标: 精确对准着陆点
- 策略: 平滑过渡，减少振荡
- 控制: 中等增益，优化轨迹

**着陆阶段 (高度 ≤ 100m):**
- 目标: 安全着陆
- 策略: 降低摆角，提高稳定性
- 控制: 低增益，小摆角，高推力

### 3. 算法实现

#### 3.1 核心函数

```python
def update_state(state, control, dt):
    # 计算推力
    thrust = throttle * thrust_max

    # 计算质量流量
    mdot = thrust / (Isp * g0)

    # 更新质量
    mass -= mdot * dt
    fuel -= mdot * dt

    # 推力分量
    thrust_x = thrust * sin(theta + gimbal)
    thrust_y = thrust * cos(theta + gimbal)

    # 加速度
    ax = thrust_x / mass
    ay = thrust_y / mass - g

    # 角加速度
    alpha = thrust * gimbal * L/2 / I

    # 更新状态
    x += vx * dt
    y += vy * dt
    vx += ax * dt
    vy += ay * dt
    theta += omega * dt
    omega += alpha * dt

    return new_state
```

#### 3.2 着陆条件检查

```python
def check_landing(state, target):
    dx = abs(state.x - target.x)
    dy = abs(state.y - target.y)

    vx_ok = abs(state.vx) < 0.5
    vy_ok = abs(state.vy) < 0.5
    theta_ok = abs(state.theta) < 0.035  # 2°
    height_ok = dy < 1.0

    return dx < 5.0 and height_ok and vx_ok and vy_ok and theta_ok
```
